Problem Statement


How can we redesign the QUBES Hub website to provide better visibility of resources and promote the QUBES mission to visitors?

Problem Description


The QUBES Hub website is a resource for education, research, and collaboration in quantitative biology. It hosts various resources like teaching materials, models, and data, which come from contributors in the community, and are used by visitors from all over the world. The QUBES team wanted to improve the user experience of the website by redesigning the website with a simplified user interface that would allow researchers to find resources easily. The new interface would better promote what the website can do for a new visitor and deliver the websiteâ€™s features and functions more efficiently.

Project Details


Duration:
May 2017 - August 2017

Role:
UX Researcher, UX Designer

Key Takeaways


  • Remote usability tests can be hampered by technical difficulties and it is important to have contingency plans for such situations.
  • Websites built on content management systems are constrained by the limitations of the platform. It is challenging to find solutions that are in line with the design recommendations while managing technical limitations.

Proposed Solution


We recommended changes to the QUBES Hub website in terms of improved visual design, clearer navigational structure, and search efficiency. To help the QUBES team realize these recommendations, we created low fidelity mockups of the proposed design and suggested further changes to the design based on user feedback.

Process


As part of my summer internship with the Science Gateways Community Institute, my colleague and I consulted with QUBES to help improve the usability of their website QubesHub. We conducted comparative analysis and heuristic evaluations to identify usability problems and understand industry conventions for features present in the QUBES Hub website.

We conducted two rounds of usability tests, the first one with the current website. Based on this study, and findings from previous user research, we created mockups for a redesigned website and tested it out with users in the second round of usability tests.

Heuristic evaluation

Using Nielsen's usability heuristics, we evaluated the QUBES Hub website and found several usability violations. We categorized our findings by key sections and tasks within the website. Some key findings from our heuristic evaluation are outlined below.

Category Problem identified Recommended fix
Homepage Too many paginations in the main banner area Display only relavent information which highlights what QUBES does or showcases recent events.
Using red color for both titles and hyperlinks confuses and distracts users Text color should be black and micro-interactions like “underline on hover” can be used to indicate the presence of links.
Navigation Breadcrumbs do not match menu structure Provide breadcrumbs properly according to the menu structure
Broken links and missing subpages in menu Fix broken links and ensure that all important pages are reachable from the menu.
Create account and Login Contrast between text and background is too low in forms Improve contrast between text and background to increase readability
Too many steps required for signing in with username and password Combine the Google sign-in and email sign-in pages so user can either use their Google account or enter their login details on the same page.
Resources section There are limited filter options for search results. Allow users to filter resources using multiple criteria like date, rating, author, etc., instead of only keywords.
No clear call to action on the subpages in Resources section Change the stylings of the “Browse resources” and “Submit resources” buttons.
Community section The tile layout of items makes it harder to find information, due to a lack of organization Organize the tiles based on category or topic to make it easier for users to find content.
Dashboard Hyperlinks in different sections of the dashboard are styled differently Keep a consistent visual style throughout the website in order to avoid confusion.

Comparative Analysis

We compared the QUBES Hub website with 5 different websites that perform similar functions and evaluated each website on ease of use, navigation, visual style and brand identity. This allowed us to determine strengths and weaknesses of the website, and understand industry standards and benchmarked best practices regarding Based on our comparative analysis, we compiled a list of changes the QUBES team should implement.

RECOMMENDATIONS

  • Concisely explain what QUBES is, the main features and benefits for members using graphics and eye-catching slogans rather than long paragraphs of text.
  • Create a distinct typographic and color hierarchy which establishes the order of importance of different content, making it easier to read and navigate through the page.
  • Provide prominent buttons and links for important functions within the site, like the search bar and login buttons.
  • Provide better filtering of search results so users can get to the content faster.
  • Place sitewide links in the footer for quicker access.

Remote Usability Testing

We conducted remote usability tests with three users of QUBES Hub website. We began by asking the participants questions about their familiarity with the website and their motivations for using the website. We then asked the participants to perform a series of tasks while thinking aloud. At the end of the tasks, we asked the participants for their general impression of the website, and comments or suggestions about the user experience.

KEY FINDINGS

  • Participants skip the homepage and go directly to their dashboard as there is too much content on the homepage, which overwhelms them.
  • Participants had trouble noticing the call-to-action buttons on pages since these buttons were not prominent.
  • Participants felt that the search results were not satisfactory as there were often too many results, and no way to sort or filter them.

Wireframing

Based on best practices from the competitive analysis result and findings from the user testing, we designed new design mockups in Axure. The wireframes were used to present design ideas to the QUBES staff for feedback, and test the ease of use of the new design with users before implementation.

In-person Usability Test

The second round of usability tests was conducted at the “Making Meaning through Modeling: Problem-solving in Biology” workshop held at Michigan State University. Six workshop attendees were asked to participate in the usability tests and asked to evaluate the mockups of the new design of the website. Each usability test was attended by one moderator, one observer and one external evaluator.

KEY FINDINGS

  • Participants were more inclined towards the search bar rather than browsing through the website.
  • Participants felt that listing resources under broad categories of Biology and Math might become overwhelming, and expressed the need for a more specialized categorization.
  • Participants were unsure how the different functions within QUBES could be useful to them, and hence were hesitant to use them.
  • Participants were overwhelmed by the complexity of the website, including new terminology and features with overlapping functionality.
